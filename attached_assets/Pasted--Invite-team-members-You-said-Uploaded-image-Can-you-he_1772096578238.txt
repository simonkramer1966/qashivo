Yep — from your screenshot the real issue showing up in logs is the Xero refresh failing with HTTP 400:

That kind of failure is usually invalid/expired refresh token (e.g. revoked consent, rotated token not stored correctly, tenant disconnected in Xero, wrong client creds, etc.). Even if the app keeps running, this can also create a repeated failure loop (scheduler/health check) that sometimes causes Replit to kill/restart the process.

Here’s what I’d do (fast, trust-safe, and prevents silent crashes):

1) Stop token-refresh failures from ever taking the server down
Goal: token problems should mark the tenant disconnected + disable sync, not throw in a way that can bubble up.

Patch: refreshAccessToken() in server/services/xero.ts (around line ~356)
Change behavior on 400 from “throw” → “gracefully disable & require reconnect”.

What to implement:

If refresh returns 400, treat as invalid_grant / revoked / expired

Set in DB:

emailConnectionStatus equivalent for Xero: xeroConnectionStatus = 'DISCONNECTED'

xeroAutoSyncEnabled = false

clear tokens (xeroAccessToken, xeroRefreshToken, xeroTokenExpiresAt)

Return null (or a typed failure result) instead of throwing

Pseudo-code shape:

try {
  const res = await axios.post(...refresh...);

  // save new tokens (including rotated refresh token!)
  await db.tenants.update({
    xeroAccessToken: res.data.access_token,
    xeroRefreshToken: res.data.refresh_token, // IMPORTANT: rotated
    xeroTokenExpiresAt: new Date(Date.now() + res.data.expires_in * 1000),
    xeroConnectionStatus: "CONNECTED",
  });

  return res.data.access_token;
} catch (err: any) {
  const status = err?.response?.status;

  if (status === 400) {
    await db.tenants.update({
      xeroConnectionStatus: "DISCONNECTED",
      xeroAutoSyncEnabled: false,
      xeroAccessToken: null,
      xeroRefreshToken: null,
      xeroTokenExpiresAt: null,
    });

    logger.warn({ tenantId, status }, "Xero refresh failed (400). Marked disconnected; requires reconnect.");
    return null;
  }

  logger.error({ tenantId, status, err }, "Xero refresh failed (non-400).");
  throw err; // keep throwing for truly unexpected cases
}
Why this matters: it prevents a bad token from creating a repeating “crash loop” and gives you a clean “Reconnect Xero” state in UI.

export async function runHealthChecks() {
  try {
    const tenants = await getTenantsWithXero();

    const results = await Promise.allSettled(
      tenants.map(t => checkTenantConnection(t))
    );

    const failed = results.filter(r => r.status === "rejected");
    if (failed.length) logger.warn({ failed: failed.length }, "Some Xero health checks failed.");

    logger.error(e, "Xero health check run failed (non-fatal).");
  }
}

4) Confirm you’re storing the rotated refresh token

refresh_token (new one every refresh)

expires_in

If you only save access_token, you’ll get exactly what you’re seeing.
