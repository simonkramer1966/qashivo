SCORING INPUTS (DETERMINISTIC)
For each debtor/contact, compute from invoice+payment history:
- Use invoices with a due_date and fully paid status (or fully settled amount_due = 0 with payment date).
- "days_late" = payment_date - due_date (negative or zero means on-time/early).
Metrics to compute (trailing window: prefer last 12 months; if less, use what exists):
- on_time_rate = % paid invoices with days_late <= 0
- avg_days_late = average max(0, days_late)
- p90_days_late = 90th percentile of max(0, days_late)
- late_30_plus_rate = % paid invoices with days_late > 30
- volatility = std dev of max(0, days_late)
- payment_recency = days since last payment
- (optional) dispute_flag_rate if you have disputes in Qashivo outcomes (Qashivo-originated), not from Xero

SCORE OUTPUT (0–100)
Implement a transparent weighted scoring model:
- Start at 100, subtract penalties based on lateness + volatility + recency.
Example (tuneable constants in one file):
- penalty_avg = clamp(avg_days_late / 60, 0..1) * 35
- penalty_p90 = clamp(p90_days_late / 90, 0..1) * 25
- penalty_late30 = late_30_plus_rate * 25
- penalty_vol = clamp(volatility / 30, 0..1) * 10
- penalty_recency = clamp(payment_recency / 120, 0..1) * 5
score = round(clamp(100 - sum(penalties), 0, 100))
Store the factors and computed metrics for explainability.

- RISKY (< 50): FIRM (tighter cadence; earlier review/escalation; consider voice under supervision if enabled)
