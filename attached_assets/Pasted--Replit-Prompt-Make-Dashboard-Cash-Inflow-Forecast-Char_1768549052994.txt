## 2) API Response Shape

Return totals + weighted totals, and also tier breakdown for tooltips:

```ts
type CashInflowPoint = {
  date: string; // ISO date for bucket start
  expectedAmount: number; // total expected cash-in (sum of remaining)
  confidenceWeightedAmount: number; // sum(remaining * confidence)

  // optional but recommended for tooltip breakdown
  highConfidence: number;   // confidence >= 0.7
  mediumConfidence: number; // 0.4 <= confidence < 0.7
  lowConfidence: number;    // confidence < 0.4

  invoiceCount: number;
};

type CashInflowResponse = {
  rangeDays: number;
  bucket: "day" | "week";
  points: CashInflowPoint[];
  asOf: string; // ISO timestamp
};
3) Confidence & Expected Date Logic (Deterministic MVP Rules)
Compute forecast from open invoices only:

status IN ('pending', 'overdue')

remaining = amount - amountPaid

skip if remaining <= 0

For each invoice determine:

expectedDate

confidence (0–1)

A) If a recent outcome/intent exists on related Actions (use what exists in the codebase)
Use latest relevant intent for that invoice/contact.

Outcome/Intent	Expected Date	Base Confidence
promise_to_pay with date	promise date	0.85
payment_made confirmation	today + 2 days	0.90
request_more_time	today + 14 days	0.55
dispute	exclude from forecast	n/a

B) Fallback (no outcome)
Condition	Expected Date	Base Confidence
invoice overdue	today + 7 days	0.35
due in future	dueDate	0.50

Risk band adjustment (Contact.riskBand)
Adjust confidence after base confidence:

A (Low): +0.10 (cap 0.95)

B: +0.05

C: +0.00

D: -0.10

E (High): -0.15 (floor 0.10)

4) Bucketing & Aggregation
Support bucket=day and bucket=week

Range is 30/60/90 days into the future from “today” (tenant timezone if available, otherwise server)

Group each invoice into the bucket containing its expectedDate

Include empty buckets so the chart remains continuous

Per bucket sum:

expectedAmount += remaining

confidenceWeightedAmount += remaining * confidence

Tier sums:

if confidence >= 0.7 → highConfidence += remaining

else if confidence >= 0.4 → mediumConfidence += remaining

else → lowConfidence += remaining

invoiceCount += 1
