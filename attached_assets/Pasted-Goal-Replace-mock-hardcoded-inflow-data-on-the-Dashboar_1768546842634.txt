2) Add a Backend Endpoint

Create a new route module if you have modular routing, otherwise add minimally to existing routes:

GET /api/dashboard/cash-inflow?range=30|60|90&bucket=day|week

Returns:

type CashInflowPoint = {
  date: string; // ISO date for bucket start
  expectedAmount: number; // sum of invoice expected amounts
  confidenceWeightedAmount: number; // sum(expectedAmount * confidence)
  invoiceCount: number;
};

type CashInflowResponse = {
  rangeDays: number;
  bucket: "day" | "week";
  points: CashInflowPoint[];
  asOf: string; // ISO timestamp
};

3) Implement the Forecast Calculation (Simple + Explainable)

Compute expected cash-in from open invoices only:

Include invoices where status IN ('pending','overdue')

remaining = amount - amountPaid (skip if remaining <= 0)

For each invoice, determine:

expectedDate

confidence (0–1)

Use this logic (MVP-grade and deterministic):

A) If invoice has an outcome/intention recorded (existing fields on Action/Invoice):

PROMISE_TO_PAY with a date → expectedDate = promiseDate, confidence = 0.85 (adjust by contact riskBand)

REQUEST_MORE_TIME → expectedDate = today + 14 days, confidence = 0.55

DISPUTE → exclude from forecast (or include with confidence 0 and amount 0); also count separately if you track it

PAYMENT_MADE_CONFIRMATION → expectedDate = today + 2 days, confidence = 0.9

NO_RESPONSE / UNKNOWN → fall through to B)

B) Otherwise fallback to due-date-based heuristic:

if overdue: expectedDate = today + 7 days, confidence = 0.35

if due in future: expectedDate = dueDate, confidence = 0.5

Risk adjustment (optional but quick):

If contact.riskBand exists:

LOW: +0.10 confidence (cap at 0.95)

MED: +0.00

HIGH: -0.15 (floor at 0.10)

Note: Use whatever intent/outcome fields actually exist in your codebase. If outcomes are on Action (latest inbound/outbound), use the most recent relevant event per invoice.

4) Bucket & Aggregate

Create buckets for the selected range (30/60/90 days)

Bucket by:

day: group by date

week: group by Monday-start week

Sum:

expectedAmount += remaining

confidenceWeightedAmount += remaining * confidence

invoiceCount += 1

Ensure output includes empty buckets (0 values) so the chart is continuous.

Top 10 invoices contributing to next 14 days

Show: invoiceNumber, debtor, remaining, expectedDate, confidence, reason (Promise / Due / Overdue fallback)

This makes conservative accountants trust it.
