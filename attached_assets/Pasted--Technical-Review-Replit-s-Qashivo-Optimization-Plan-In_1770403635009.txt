## Critical Questions & Recommendations

### 🔴 CRITICAL: Verify Before Proceeding

#### 1. Foreign Key Constraints
**Question:** Do any of the 38 tables have foreign keys pointing TO them from tables being kept?

**Action Required:**
```sql
-- Run this for each table in Tier 1 & 2 before deletion
SELECT 
  tc.table_name AS referencing_table,
  kcu.column_name AS referencing_column,
  ccu.table_name AS referenced_table,
  ccu.column_name AS referenced_column
FROM information_schema.table_constraints tc
JOIN information_schema.key_column_usage kcu 
  ON tc.constraint_name = kcu.constraint_name
JOIN information_schema.constraint_column_usage ccu 
  ON ccu.constraint_name = tc.constraint_name
WHERE tc.constraint_type = 'FOREIGN KEY'
  AND ccu.table_name IN ('userPermissions', 'userInvitations', ...);
```

**Risk if skipped:** Cascade deletes or foreign key constraint violations

**Recommendation:** Add this as "Phase 0" - run FK check, document results before any deletions

## Red Flags to Watch During Implementation

### Stop Implementation If:
1. ❌ Any Tier 1 table deletion fails with FK constraint error
2. ❌ After merging services, >5% of unit tests fail
3. ❌ After N+1 fixes, any query takes >2x longer than before
4. ❌ After lazy loading, any page shows blank screen >1 second
5. ❌ Database size increases after adding indexes (should stay same or decrease)

### Proceed with Caution If:
1. ⚠️ Any Tier 2 table has >3 code references (needs deeper audit)
2. ⚠️ Service consolidation affects >10 import statements
3. ⚠️ Index creation takes >10 minutes (might indicate data issues)
