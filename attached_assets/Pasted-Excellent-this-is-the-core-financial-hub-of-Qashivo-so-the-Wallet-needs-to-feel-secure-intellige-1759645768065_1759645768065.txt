💠 QASHIVO WALLET — UI / UX DESIGN SPEC
🎯 Core Purpose

A unified financial dashboard that lets users:

Receive incoming payments (customer, insurer, lender).

View and manage their current wallet balance and transactions.

Make outgoing payments (insurance premiums, supplier payouts, transfers).

Control invoice-related flows (finance drawdown, insurance coverage, repayment).

Integrate seamlessly with accounting systems (e.g., Xero).

🧩 PAGE STRUCTURE
1. Header / Summary Bar (Sticky Top)

Purpose: Give instant visibility to available funds and recent activity.
Layout:

Wallet Balance: Prominent, large font (“Available Balance £12,450.22”)

Pending Funds: Subtext (“£1,200 incoming | £300 outgoing”)

CTA Buttons:

➕ Top Up / Deposit

🔄 Withdraw / Transfer

⚙️ Settings

Micro-interaction: Animate balance updates with subtle pulse or slide-in when funds change.

2. Tabbed Navigation

Below header:

[ Overview ] [ Transactions ] [ Invoices ] [ Insurance ] [ Finance ]


Each tab filters or expands the view based on purpose.

🧭 TAB DETAILS
🟢 Overview Tab

Purpose: Snapshot of all activity in one glance.

Sections:

Balance Graph (Line or Area Chart):

Shows Wallet balance over time.

Hover reveals transaction events (payments, premiums, drawdowns).

Activity Feed (Chronological Timeline):

Example entries:

💸 “£85 insurance premium paid to Allianz — Invoice #1047 (Fully insured)”

🏦 “£5,000 finance advance credited by Funding Circle”

Filter: Today | 7 days | 30 days | Custom

Summary Cards (Grid, 2x2):

Premiums Paid: £430

Quick Actions Panel:

“Withdraw to Bank”

“Export Statement”
