========================================================

type TimelineItem = {
  id: string;
  occurredAt: string; // ISO datetime (UTC)
  direction: "outbound" | "inbound" | "internal";
  channel: "email" | "sms" | "voice" | "note" | "system";
  summary: string;
  preview?: string;
  body?: string;
  subject?: string;
  participants?: { from?: string; to?: string[] };
  outcome?: {
    type:
      | "promise_to_pay"
      | "request_more_time"
      | "payment_plan"
      | "dispute"
      | "wrong_contact"
      | "paid_confirmed"
      | "refused"
      | "no_response"
      | "other";
    confidence: number; // 0..1
    extracted?: Record<string, any>;
    requiresReview?: boolean;
  };
  status?: "sent" | "delivered" | "failed" | "received" | "transcribed";
  externalRefs?: {
    provider?: "sendgrid" | "vonage" | "retell" | "stripe" | "xero";
    providerMessageId?: string;
  };
  createdBy?: { type: "system" | "user"; name?: string; userId?: string };
};

TIMELINE UI RULES (CALM + PROGRESSIVE DISCLOSURE)
1) Default:
- newest first
- load last 50 items with “Load more” at bottom
- optional quiet date separators: Today / Yesterday / DD Mon

2) Row anatomy:
- subtle channel glyph + direction indicator
- one-line summary (required) as primary text
- secondary line: relative time + muted status
- optional outcome label (muted text, not bright badge)
- No heavy borders/cards; hairline dividers and whitespace

3) Confidence display:
- NEVER show numeric confidence by default
- Map:
  - >=0.85 => “High confidence”
  - 0.65–0.84 => “Medium confidence”
  - <0.65 OR requiresReview => “Needs review”

4) Hover:
- reveal subtle “View” affordance
- optional tooltip preview (max 240 chars) if preview exists; preserve line breaks

5) Click:
- opens a right-hand item detail drawer with:
  - channel + direction + exact timestamp
  - subject/body/transcript/note body (plain text or safely sanitized)
  - participants (from/to)
  - outcome panel (if present):
    - human label for outcome type
    - extracted entities (e.g., promised date)
    - confidence label (High/Medium/Needs review)
    - if requiresReview: show “Needs review” (neutral)
  - collapsed “Details” disclosure for provider refs/status IDs

6) Filters:
- hidden by default
- small “Filter” text button opens a POPOVER with:
  - Channel: Email / SMS / Voice / Notes
  - Direction: Inbound / Outbound / Internal
  - Toggle: “Show outcomes”
  - Toggle: “Needs review only”
- No search bar; no sidebar

7) Notes:
- created as TimelineItem channel="note" direction="internal"
- displayed inline in stream with “Note” label (muted)

8) System events:
- include ONLY meaningful state changes:
  - chasing paused until X
  - channel disabled due to opt-out
  - escalated for review
  - payment detected in Xero (partial/full)
  - contact role updated
- Do not include noisy micro-events

9) Accessibility:
- rows focusable, Enter opens drawer, Esc closes
