3) Add Collection Policy (timers + cadence) — must do
3.1 Schema migration

3.2 Apply policy to actions
On send: set awaitingReplyUntil = sentAt + waitDaysForReply


On no reply: set cooldownUntil = now + cooldownDaysBetweenTouches


On max touches: set escalation flag and/or create attention_item type ESCALATION_DUE


Acceptance
A scheduled job (or request-time evaluator) moves items between “awaiting reply / cooldown / escalation due”.



4) WorkState / InFlightState (don’t rewrite actions yet) — layer it
4.1 Minimal fields added to actions (or a view)
Instead of replacing status + approvalStatus, add two columns:
workState (PLAN|IN_FLIGHT|ATTENTION|RESOLVED)


inFlightState (SCHEDULED|SENT|AWAITING_REPLY|COOLDOWN|ESCALATION_DUE|DELIVERY_FAILED)


Default mapping:
approvalStatus=PENDING => workState=PLAN


approvalStatus=APPROVED and status in {QUEUED,SENT} => workState=IN_FLIGHT


if attention_item exists open => workState=ATTENTION


if invoice paid => workState=RESOLVED


4.2 Transition writer (single function)
Create setWorkState({invoiceId, debtorId, workState, inFlightState, reason})
updates action(s)


writes AuditEvent STATE_CHANGED


optionally creates/resolves attention_items


Acceptance
Every movement between Plan/In-Flight/Attention is recorded as an event.



5) Outcome routing rules (the “Interpret → Route” heart)
Implement routeFromOutcome(outcome):
Routing:
If confidence < 0.65 ⇒ Attention: LOW_CONFIDENCE_OUTCOME


If type in:


DISPUTE, DOCS_REQUESTED, CONTACT_ISSUE, CANNOT_PAY, PAID_ALREADY_CLAIM, REQUEST_CALL_BACK, BANK_DETAILS_CHANGE_REQUEST
 ⇒ Attention (create attention_item with subtype = outcome.type)


If type in:


PROMISE_TO_PAY, PAYMENT_IN_PROCESS
 ⇒ remain IN_FLIGHT (await payment) + schedule confirmation action (confirmPTPDaysBefore)


If PAYMENT_PLAN_PROPOSED:
 ⇒ Attention (unless you add “accept plan” later)


If NO_RESPONSE / AMBIGUOUS:
 ⇒ remain IN_FLIGHT, apply cooldown rules


Also implement payment routing:
PAYMENT_SIGNAL paid in full ⇒ RESOLVED


part-paid ⇒ update remaining + return to PLAN (or IN_FLIGHT depending on policy)


Acceptance
Simulated inbound reply creates Outcome → creates attention_item or updates in-flight timers and forecast.

PAYMENT_IN_PROCESS: window midpoint/latest (MED)


Else heuristic (LOW)


6.2 Update forecast_points

confidenceWeightedAmount
