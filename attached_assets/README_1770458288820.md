# Demo Xero Integration System

A complete demo Xero API simulator that generates realistic business data and flows through your existing Qashivo integration middleware without modification.

## 🎯 What This Does

- **Generates realistic demo companies** with different business archetypes (SaaS, Construction, Retail, Manufacturing, etc.)
- **Matches Xero API formats exactly** - no changes needed to your integration code
- **Simulates live trading activity** - new invoices, payments received, etc.
- **Provides multiple customer payment behaviors** - reliable, mixed, slow, struggling
- **Works with your existing Xero sync logic** - just intercepts API calls for demo tenants

## 📁 Files Structure

```
server/services/xero/
├── xero-response-formats.ts      # Xero API type definitions
├── demoXeroCompany.ts            # Company generator with business archetypes
├── demoXeroInterceptor.ts        # API call interceptor
└── INTEGRATION_GUIDE.ts          # Detailed integration instructions
```

## 🚀 Quick Start

### 1. Copy Files to Your Project

```bash
# Place these files in your server/services/xero/ directory:
- xero-response-formats.ts
- demoXeroCompany.ts
- demoXeroInterceptor.ts
```

### 2. Initialize at Server Startup

In your `server/index.ts`:

```typescript
import { DemoXeroInterceptor } from './services/xero/demoXeroInterceptor';

// Add near the top of your startup:
DemoXeroInterceptor.initialize();
```

This creates 4 demo companies automatically:
- CloudFlow SaaS Ltd (monthly recurring, reliable payments)
- BuildRight Construction Ltd (milestone payments, slow payers)
- HighStreet Fashion Ltd (weekly orders, mixed behavior)
- PrecisionTech Manufacturing Ltd (batch orders, reliable)

### 3. Modify Your Xero Service

In your existing `server/services/xero.ts`, wrap your API calls:

```typescript
import { DemoXeroInterceptor } from './xero/demoXeroInterceptor';

export async function syncXeroInvoices(tenantId: string) {
  // Check if demo tenant
  const intercepted = DemoXeroInterceptor.interceptXeroCall(
    tenantId,
    '/api.xero.com/api.xro/2.0/Invoices',
    'GET'
  );

  if (intercepted.shouldIntercept) {
    // Use demo data
    await processXeroInvoicesResponse(tenantId, intercepted.response);
    return { success: true, count: intercepted.response.Invoices.length, source: 'demo' };
  }

  // Normal Xero API call for real tenants
  const xeroClient = await getXeroClient(tenantId);
  // ... your existing code
}
```

### 4. Add Demo Management Routes

Copy the route examples from `INTEGRATION_GUIDE.ts` to your `server/routes.ts`.

### 5. Test It!

```bash
# List demo companies
curl http://localhost:5000/api/demo/xero-companies

# Create a custom demo company
curl -X POST http://localhost:5000/api/demo/setup-xero-company \
  -H "Content-Type: application/json" \
  -d '{"tenantId": "my-demo-saas", "archetype": "SaaS"}'

# Simulate 30 days of trading
curl -X POST http://localhost:5000/api/demo/simulate-trading \
  -H "Content-Type: application/json" \
  -d '{"tenantId": "my-demo-saas", "days": 30}'
```

## 💡 Best Practices

### 1. Use Separate Tenant IDs
```typescript
// Good
createCompany('demo-saas-1', 'SaaS');
createCompany('demo-construction-1', 'Construction');

// Avoid mixing with real tenants
createCompany('real-customer-123', 'SaaS'); // DON'T DO THIS
```

### 2. Clear Demo Data Regularly
```typescript
// Clear specific tenant
DemoXeroInterceptor.clearCompany('demo-tenant-1');

// Or reset all
DemoXeroInterceptor.resetAll();
```

### 3. Use Realistic Timeframes
```typescript
// Good - test gradual behavior
simulateTradingActivity(tenantId, 90); // 90 days

// Avoid - too much data at once
simulateTradingActivity(tenantId, 365); // Generates 100+ invoices
```

### 4. Check Before Creating
```typescript
if (!DemoXeroInterceptor.isDemoTenant(tenantId)) {
  DemoXeroInterceptor.createCompany(tenantId, archetype);
}
```

## 🎨 Frontend Example

```typescript
import React, { useState, useEffect } from 'react';

export function DemoCompanySelector() {
  const [companies, setCompanies] = useState([]);
  const [selected, setSelected] = useState(null);

  useEffect(() => {
    fetch('/api/demo/xero-companies')
      .then(r => r.json())
      .then(data => setCompanies(data.companies));
  }, []);

  const simulateWeek = async () => {
    await fetch('/api/demo/simulate-trading', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ tenantId: selected, days: 7 }),
    });
    alert('Week simulated! Refresh to see new data.');
  };

  return (
    <div>
      <select onChange={e => setSelected(e.target.value)}>
        <option>Select demo company...</option>
        {companies.map(c => (
          <option key={c.id} value={c.id}>
            {c.name} - {c.stats.invoices} invoices
          </option>
        ))}
      </select>
      
      <button onClick={simulateWeek} disabled={!selected}>
        Simulate 1 Week
      </button>
    </div>
  );
}
```

## 🐛 Troubleshooting

### Demo data not appearing?
- Check server logs for "Intercepting Xero API call" messages
- Verify `DemoXeroInterceptor.initialize()` was called at startup
- Confirm tenant ID matches exactly

### Invoices not syncing?
- Ensure your `syncXeroInvoices()` function calls the interceptor
- Check that `processXeroInvoicesResponse()` handles the response correctly
- Verify database permissions

### Payment behavior not realistic?
- Review the archetype config in `demoXeroCompany.ts`
- Adjust probability curves in `shouldBePaid()` method
- Consider payment terms (30, 60, 90 days)
