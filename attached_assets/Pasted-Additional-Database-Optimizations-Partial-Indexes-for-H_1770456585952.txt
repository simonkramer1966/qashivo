Additional Database Optimizations
Partial Indexes for High-Volume Tables
sql-- For soft-deleted records (if you use soft deletes)
CREATE INDEX idx_invoices_active ON invoices(tenant_id, status) 
WHERE deleted_at IS NULL;

-- For recent data queries
CREATE INDEX idx_actions_recent ON actions(tenant_id, created_at) 
WHERE created_at > NOW() - INTERVAL '90 days';
Analyze Table Statistics

Backend Performance Wins
Connection Pool Tuning
Review your database connection pool settings. Common issues:

Pool size too small → connection exhaustion
Pool size too large → memory waste
Missing idleTimeoutMillis → zombie connections

Response Compression
Enable gzip/brotli compression in your Express middleware if not already done (typically 60-80% size reduction).
Drizzle Query Optimization

Review .select() calls - only fetch needed columns
Check for accidental .toArray() calls that load entire result sets
Consider .prepare() for frequently-run queries

Code Quality Improvements
TypeScript Strict Mode Gaps
Your tsconfig.json likely has some strict checks disabled. Consider enabling:

noUncheckedIndexedAccess - prevents array[0] undefined bugs
noPropertyAccessFromIndexSignature - catches typos in object access

Remove Dead Code
Run a dead code detector:
bashnpx ts-prune | grep -v "(used in module)"
This finds exported functions/types that nothing imports.
