🧭 Replit Prompt — “Generate Intelligent Demo Data (200 Customers with Learning History)”

Prompt:

I’m building an investor demo for Qashivo (Nexus AR) — an AI-driven accounts receivable system.
Please create a new dataset seeder called seed/seed_intelligent_demo.py that generates a learning-ready simulation for ~200 customers with 12 months of invoices and communications.

1️⃣ Purpose
I need to show investors how Qashivo’s Intent Analyst and ML models “learn” over time — so the seed data must include evolving customer behaviour and realistic messages.

2️⃣ Data Requirements

200 customers, each with:

A behavioural profile type (reliable, serial_promiser, predictable_late, disputer, deteriorating)

Unique name, contact email, and segment

For each customer:

12 months of invoices (15–30 per customer)

Invoices include: amount, issue_date, due_date, status, paid_date

Random overdue patterns per behaviour type

Communications history:

5–10 messages per invoice (mix of inbound/outbound)

Channels: email, sms, voice

Message text examples:

Reliable: “Paid as promised, thank you!”

Serial Promiser: “We’ll pay Friday” (repeated, low follow-through)

Predictable Late: “We always pay by 10th next month”

Disputer: “This invoice is incorrect”

Deteriorating: starts positive, becomes negative over months

Promise-to-Pay (PTP) history:

Randomised probability of PTP kept vs broken (based on customer profile)

Add timestamps and outcome tracking (kept, broken, active)

Sentiment drift:

Each customer’s sentiment score changes over time (e.g., -0.2/month for deteriorating)

Include computed summary metrics for each month:

Promise reliability (% kept)

Average sentiment

Response time (days)

3️⃣ Output

Tables to populate:

customers

invoices

communications

promises

customer_features_history

Save generated dataset to CSV or insert into the database (PostgreSQL or SQLite, based on env).

Print a clean summary when done:

✅ Seeded 200 customers

✅ Created 27,000 messages
✅ Recorded 9,500 Promise-to-Pay events
